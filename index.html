07_容器数据卷

发表于 2021-08-02 分类于 常用组件学习笔记 ， Docker
本文字数： 1.3k 阅读时长 ≈ 1 分钟

1. 为什么需要它

当应用程序和运行环境打包成一个容器在服务器运行的时候，我们希望它是能够持久化的，同事容器之间可以共享数据。因为 Docker 容器产生的数据如果不通过docker commit生成新的镜像，使数据作为镜像的一部分保存下来，那当容器删除后，数据自然也被删除了。

比如我启动了一个 MySQL 的容器，里面放了我的数据库，结果我把容器一删，里面的数据全没了，哦嚯。

为了能在 Docker 中保存数据，就要使用卷，让数据挂载到本地。而不是每次都要用docker exec进入容器，到处找目录来操作文件。

2. 什么是卷

「卷」指代非理性的内部竞争。指同行间竞相付出更多努力以争夺有限资源，从而导致个体「收益努力比」下降的现象……啊这是「内卷」不是数据卷。

卷就是目录或文件。它存在于一个或多个容器中，由 Docker 挂载到容器，但不属于 UnionFS，因此卷可以绕过 UnionFS，提供一些用于持续存储或共享数据的特性。卷的设计目的就是数据的持久化，它完全独立于容器的生存周期，因此 Docker 不会容器删除时删除其挂载的数据卷。

02_Docker 安装

发表于 2021-07-20 分类于 常用组件学习笔记 ， Docker
本文字数： 2.4k 阅读时长 ≈ 2 分钟

1.基本组成结构

  • 镜像Images：

    Docker 镜像就是一个只读的模板。镜像可以用来创建 Docker 容器，一个镜像可以创建很多容器。类似 Java 中的类和对象，类就是镜像，对象就是容器。

  • 容器Containers：

    • Docker 利用容器独立运行一个或一组应用。容器是用镜像创建的运行实例。
    • 它可以被启动、开始、停止、删除。每个容器都是互相隔离的。
    • 容器可以看成简易版的 Linux 环境（包括 root 用户权限、进程空间、用户空间和网络空间等）。
    • 容器和镜像几乎一模一样，区别在于容器最上层是可读可写的。

  • 仓库Repository：

    • 仓库是集中存放镜像的地方。
    • 仓库（Repository）和仓库注册服务器（Registry）是不一样的。仓库注册服务器上往往存放着多个仓库，每个仓库又包含了多个镜像，每个镜像有不同的标签。
    • 仓库分为公开仓库和私有仓库。
    • 最大的公开仓库是Docker Hub ，国内的公开仓库包括阿里云、网易云等。

01_Docker 概述

发表于 2021-07-20 分类于 常用组件学习笔记 ， Docker
本文字数： 1.6k 阅读时长 ≈ 1 分钟

1. 为什么会出现

一款产品一般会经历从开发到上线，从操作系统到运行环境，再到应用配置的流程。各种版本的迭代之后，不同版本环境的兼容，对运维人员是极大的考验。

就像重装电脑之后，所有软件就要重新安装一样，换一台机器，就要重新配一次环境，费时费力。如果软件可以带着环境一起安装，就是说在部署项目的时候，把原始环境一模一样地复制过来，就可以解决「但是在我的机器上可正常工作啊」的问题。

传统上认为，软件编码开发和测试结束后，所产出的成果即是程序或是能够编译执行的二进制字节码文件等（以 .class 文件为例）。而为了让这些程序可以顺利执行，开发团队也得准备完整的部署文件，让维运团队得以部署应用程式，开发需要清楚的告诉运维部署团队，用到的全部配置文件和所有软件环境。即便如此，仍然常常发生部署失败的状况。

Docker 之所以发展如此迅速，是因为它对此给出了一个标准化的解决方案。Docker 镜像的设计，使得 Docker 得以打破过去「程序即应用」的观念。通过 Docker 镜像 ( Docker images ) 将应用程序所需要的系统环境，由下而上打包，实现应用程序跨平台间的无缝接轨运作。

Docker 的思想来自于集装箱。在一艘大船上，可以把货物规整的摆放起来。并且各种各样的货物被集装箱标准化了，集装箱和集装箱之间不会互相影响。那么就不需要专门运送水果的船和专门运送化学品的船了。只要这些货物在集装箱里封装的好好的，那我就可以用一艘大船把他们都运走。

1. 解释

Redis 的发布订阅是一种消息通信模式。如果一个订阅者对发布者做了订阅操作，当发布者发布一个消息的时候，订阅者就可以接受消息。发布者可以在不同的频道发布消息，同样的，订阅者可以订阅任意数量的频道。

比如：当我在 b 站关注了一个 UP 主，这个 UP 在 b 站发布新视频或新动态的时候，我就能收到推送。但是如果他在 YouTube 发布了一个视频，我没有订阅他 YouTube 的频道，我就无法收到消息。
